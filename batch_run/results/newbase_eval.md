 检索线路对比实验报告：Basic vs Enhanced                                                                      
                                                                                                               
  1. 任务设计与选择理由                                                                                        
                                                                                                               
  任务： Convergence Analysis of First-Order Optimization Methods（一阶优化方法收敛性分析）                    
                                                                                                               
  为什么选这个题目：                                                                                           
                                                                  
  论文库（Mathematical Programming 期刊 3,243 篇论文，135                                                      
  篇核心标注）中覆盖最密集的领域之一是"基于概率模型的优化收敛分析"和"非凸优化算法与理论"这两个 L2
  子域。一阶优化方法横跨这两个子域，既有经典次梯度理论（Held/Wolfe/Crowder 1973, Kiwiel                        
  1985），也有近端分裂方法（Eckstein/Bertsekas 1992），还有现代复杂度分析（Bolte et al.
  2016）。这使得它非常适合测试检索管线的域路由能力——enhanced 管线的 domain-first
  检索需要在多个子域间正确分发查询。

  Writing Plan 设计了 3 个 section（Subgradient → Proximal/Splitting → Modern Complexity），明确定义了 3 个    
  gap（unification gap, relaxed assumptions gap, rate tightness gap），并列出了 10 条 Key References
  作为评测锚点。

  Basic 特点：
  - 引用少但精（22 条，全部有完整元数据）
  - Key Ref 覆盖高（70%），说明 FAISS 向量搜索直接匹配到了 Plan 中的核心文献
  - 幻觉率严重（53%），因为 basic 只返回 title+abstract，LLM 在缺乏上下文的情况下"创造"了大量文献名
  - 写作流畅，3 个 section 结构清晰，但引用深度不足

  Enhanced 特点：
  - 引用量大（58 条），bib 文件几乎是 basic 的 2 倍
  - 幻觉率极低（仅 7%），因为 enhanced 返回了丰富的上下文（算法卡片、定理卡片等），LLM 有充足信息避免编造
  - Key Ref 覆盖极低（仅 20%），只命中了 Kiwiel 1985 和 Nesterov 2007
  - BibTeX 质量问题：83% 的条目缺少 author 字段
  - 每个 section 引用了大量来自域内检索的论文，但这些论文并非 Plan 指定的核心文献

  3. 分数差距分析

  为什么 Enhanced 在内容准确性上大幅领先（7.82 vs 4.19）：

  根本原因是信息丰富度差异。Basic 管线只返回论文的 title 和 abstract（约 200-300 字/篇），LLM                                   
  缺乏足够信息来准确描述论文贡献，于是频繁"编造"看起来合理但实际不存在的文献。Enhanced
  管线返回了域层级结构、算法卡片、定理卡片和 PageRank 排序后的丰富上下文，LLM 能够基于实际内容进行写作。                        
                                                                  
  为什么 Basic 在引用相关性上反而更高（6.05 vs 5.20）：                                                                         
   
  这个逆转揭示了 enhanced 管线的一个结构性缺陷：                                                                                
                                                                  
  1. Key Ref 覆盖 70% vs 20% — Basic 的 FAISS 全局搜索以整个 query 做向量匹配，恰好命中了 Plan 中列出的经典文献（Held 1973,     
  Eckstein 1992, Bolte 2016 等）。Enhanced 的域路由将查询分配到特定子域（如"基于概率模型的优化收敛分析"），但 Plan 中的 Key
  References 分散在多个子域中，域路由可能将某些经典文献"路由偏了"——比如 Held/Wolfe/Crowder 1973 的 subgradient optimization     
  可能被路由到了不包含该文的子域。                                
  2. 引用质量 vs 引用数量 — Enhanced 产出了 58 条引用但大部分是"弱相关"（9/15 weak），说明域内检索返回了大量同域但不精确对应
  Plan 需求的论文。Basic 的引用虽少，但非幻觉部分全部"强相关"（7/14 strong, 0 weak）。                                          
  3. BibTeX 元数据缺失 — Enhanced 的 bib 条目 83% 缺少 author 字段，这是因为域检索返回的论文信息中 author
  字段可能被省略或格式不同，导致 LLM 无法正确填充。这进一步拉低了引用质量得分。                                                 
                                                                  
  总结：Enhanced 管线在"不编造"方面显著优于 Basic（幻觉率 7% vs 53%），但在"找对文献"方面不如 Basic 的全局搜索。                
  理想方案应该是：用 Enhanced 的丰富上下文来保证准确性，同时用 Basic 的全局 FAISS 搜索来确保 Key Reference
  覆盖，即两条管线的融合。
